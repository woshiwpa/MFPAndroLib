v 1.8.0

1. Start to support parallel computing via call statement. This is a mile stone of MFP, a Mathematical language For Parallel computing. With this new feature, user using a mobile device may trigger a sophisticated code block, e.g. a game or an algorithm, running in a different device. At this moment parallel computing is still in the testing phase so bugs are inevitable. Also only TCP/IPv4 is supported. More protocols will be added gradually.

2. Start to support concurrent computing, i.e. multi-threading, also via call statement. In this way, a unified interface is provided for both parallelization and concurrency.

3. Provide a signaling function suspend_until_cond to synchronize (inter-process) threads.

4. Fix many bugs for Android N and above versions;

v 1.7.4

1. Support data/audio/video communication (still in testing phase) and detailed manual has been added since 1.7.4.70. Please note that different from WhatsApp or Line, Scientific Calculator Plus is not a chatting app. Data/audio/video communication support will be the foundation of parrallel computing, a milestone of Scientific Calculator Plus and MFP programming language. To this end, Scientific Calculator Plus implements data/audio/video communication in a very different way. It is absolutely P2P (peer to peer). Peer id is email address. Also, a side-benefit from P2P is that a remote monitoring system can be easily setup using two Android phones and this app.

In order to test this functionality, user may start Scientific Calculator Plus, select Settings, then select Remote connection settings, then click OK. A dialog box would be populated asking for email address, password, SMTP server address and port, and IMAP server address and port. It is recommended to use Microsoft Outlook or Hotmail. If user inputs an Microsoft Outlook or Hotmail email and its password, the software will auto-complete SMTP and IMAP server addresses and ports. Google Gmail is also supported. However, Gmail client app login password, which is also used here, may be different from web login password. User has to check Gmail settings and figure out the right password. Like Microsoft emails, the software can auto-complete SMTP and IMAP settings. Besides Outlook, Hotmail and Gmail, other email addresses are also supported as long as they support email client apps and do not have very harsh spam filter. But Scientific Calculator Plus cannot auto-complete SMTP and IMAP settings for them.

After all the settings are filled, user may tap OK button to save and waiting for incoming connect requests. Instead, if user taps Test, a dialog will be popped out asking for remote email address. User, in this case, should type in the email address of communication peer, or input nothing to start a local echo session. Then user taps OK and waits for the peer to start (if it is not a local echo session). The other party should do the same settings, allow service to start communication session, and then tap OK button to wait for incoming request. Around 5 minutes after test session is started user will see video connection ready. By then user is able to talk and send messages to the remote peer.

Now the GemGem game can be played in even very old Android devices. However, user may still complains that in Android devices, MFP is much slower than other scripting languages. The reason is MFP uses BigDecimal to ensure calculation accuracy while other scripting languages generally choose double, which is much less accurate and much faster. BigDecimal hasn't been optimized in Android and it's around 1000 times slower than double. To cope with this issue, MFP will introduce double type in the future;

v 1.6.6

1. From 1.6.6.50, Scientific Calculator Plus is able to compile any mfps file with a unicode name. This means when creating an APK package, Scientific Calculator Plus no longer copies all user defined .mfps files. Instead, it extracts related .mfps files which includes all referred functions. In some situations, e.g. calling integrate or plot_exprs function, parameter is a string or string based variable so that Scientific Calculator Plus is not able to identify which function will be called in the run at compiling time. User, therefore, needs to add an annotation, @complusory_link, before the calling statement explicitly telling Scientific Calculator Plus what functions should be linked into package. For instance

...

@compulsory_link function(myf_expr1, myf_expr2(2))

integrated_result = integrate(expression_str, variable_str)

...

In this example, myf_expr1 and myf_expr2 are user-defined functions which may be called by function integrate to be integrated. All the functions with name myf_expr1 will be compiled into APK package, no matter how many parameters they have. However, only function whose name is myf_expr2 and who has exactly 2 parameters or has optional parameter(s) are compiled into APK package.

If user wants to include all user defined functions, please use annotation statement

@compulsory_link all_functions

. Scientific Calculator Plus will link-in all the functions. However, this means that user has to ensure that all referred functions have been defined. Otherwise, there will be compiling error.

Also note that the @compulsory_link annotation should be located inside the body of the called function. If it is above function statement or below endf statement, it will not take any effect.

2. MFP will change the behavior of set_array_elem function. At this moment, set_array_elem will change the content of its first parameter anyway, even if its first parameter is a simple value instead of matrix. From 1.7.x all the parameters which are not matrix cannot be modified inside a function body. So if user writes a statement like

set_array_elem(x, [1,2], 2+3i)

And expect x to be updated after this statement. Please change it to

x = set_array_elem(x, [1,2], 2+3i)

User can use both the old way, i.e. set_array_elem(x, [1,2], 2+3i) and the new way, i.e. x = set_array_elem(x, [1,2], 2+3i) to update x value in any 1.6.x version from now. We hope this compability gives enough time for user to make change.

3. Scientific Calculator Plus now is able to build independent APK (Android app installation package) file from an MFP function, whether it is a built-in function or a user defined function. User can set Android app name, package id, app icon for the APK file, and can also select which function parameter(s) to input and config app manual. v 1.6.5

1. 2D graph engine for expressions is improved in 1.6.5.48. Caching technology is implemented to store calculated results. And the expression chart is more sensative to Drag-to-move operations. However, note that plotting expressions is different from plotting data. Expression result at a new value need be calculated anyway. As such if the expression is not simple, user may still experience significant lag.

2. Plot graph for 2-variable and 3-variable implicit functions. Note that only solvable implicit functions can be plotted. Polymonials of degree larger than 3 can be solved but cannot be plotted because the root expressions are very complicated. If plot 2-variable implicit function, at most 4 root expressions can be drawn. For 3-variable implicit function, Calculator may calculate roots for each of the variables and then plot at most 2 roots for every variable. In other words, a single 3-variable implicit function may imply at most 6 to-be-plotted expressions. The whole solving-plotting procedure in this case will take quite long time. Also note that implicit function must be an equation, not assignment, i.e. x**2+y**2=4-z**2 is invalid while x**2+y**2==4-z**2 can be accepted;

3. Change behavior of functions size, zeros and ones. Now if calculate size of a value, not matrix, [] not [0] will return. Also, function zeros and ones return 0 and 1 respectively if input parameter is [];

v 1.6.0

It is strongly recommended that user does not run more than 6*6 matrix eigen values/vectors calculation in a mobile device and does not run more than 8*8 matrix eigen values/vectors calculation in a PC.
