程式開發科學計算器的新特性

程式開發科學計算器已經在百度貼吧安家了，請到百度MFP吧參加對程式開發科學計算器以及MFP程式開發語言的討論以獲得最快捷的幫助和最新的資訊。

2.1.0版

1. 開始在安卓系統上支持基於WebRTC的多媒體庫，這樣遊戲玩家就可以邊玩遊戲邊視頻聊天了。

2. 開始支持雙向通信連接。

3. 開始支持JSON格式的文件的讀寫。

2.0.1版

1. 開始在安卓系統上支持基於電子郵件地址的WebRTC通信協議，以實現點對點通訊和程式開發。

2.0.0版

1. 開始支持面向對象的程序設計，也就是聲明和管理類和對象。

1.8.1版

1. 增加了支持並行計算的消息函數send_sandbox_message和read_sandbox_message。

2. 增加了更多並行計算的示例，包括一個采用消息同步機制的多玩家貪吃蛇的遊戲，以及采用call程序塊參數進行同步的分布式人工神經網的例子。

3. 修正了很多錯誤。

1.8.0版

1. 增加了對並行計算的支持。事實上，MFP語言就是Mathematical language For Parallel computing的縮寫。這種新的功能允許用戶通過自己的手機或者任何支持MFP語言的設備在另外一個支持MFP語言的設備上運行一段代碼（比如，一種算法或者一個遊戲）。需要註意的是這一部分現在還是處於測試階段。僅僅只有TCP/IPv4協議被支持，並且不排除有錯誤。但是對更多協議的支持會逐步被加入，MFP語言也會更加完善。

2. 增加了對多線程的支持。並且多線程計算和並行計算采用的是同樣的程式界面，大大降低了開發的難度。

3. 提供了信號函數程式suspend_until_cond，該函數程式可以實現（跨進程）線程同步。

4. 修正了很多錯誤，現在本軟體可以在安卓N及其以上版本穩定運行。

1.7.4版

1. 增加了通過email進行數據，語音和視頻通信的功能（還處於測試階段）並且從1.7.4.70版開始增加了詳細的使用文檔。需要註意的是，程式開發科學計算器引入的這些功能和微信、QQ不同。微信和QQ本質上是通過客戶機-伺服器的方式實現數據，語音和視頻通信，而程式開發科學計算器則是真正支持端到端（Peer-to-Peer）的，去中心化的通信。當然，程式開發科學計算器引入的這些功能並非是為了和微信、QQ等通信軟體競爭，這些功能的引入是為了實現真正意義上的分布式計算。換句話說，引入這些功能將成為程式開發科學計算器和MFP程式語言發展的一個裏程碑。端到端通信的另外一個好處是，架設一個遠程監控系統可以用兩部安卓手機加本應用輕松實現。

由於還處於非常初級的測試階段，端到端通信的功能現在僅能收發短消息和進行語音和視頻的對話。用戶如果想測試其功能，請在打開程式開發科學計算器之後，點擊“設置”，選擇“遠程連接設置”，然後點擊確定，進入遠程連接設置，然後輸入Email郵箱地址和密碼。推薦Microsoft的outlook或者hotmail的郵件地址，這樣用戶就不用輸入端口和伺服器地址，軟體會自動補全。也可以使用google的gmail或騰訊的qq郵箱，但需要註意的是，由於采用的客戶端的方式讀和發送郵件，此時gmail或qq郵箱的密碼和通過網頁登陸郵箱的密碼不同。Gmail或qq郵箱的端口和伺服器地址，軟體也可以自動補全。如果用戶使用其他郵箱，從理論上也是可以的，但是需要註意的是該郵箱必須支持客戶端收發郵件，同時不能夠有太嚴格的垃圾郵件過濾機制，另外用戶還必須自己填寫伺服器地址和端口。

設置填寫完畢之後，用戶如果點擊確定，可以將設置保存，如果點擊測試則彈出輸入文本的對話框，要求輸入對方郵件地址。如果不輸入地址，軟件將會嘗試連接自己。然後點擊確定，等待對方（對方也需要進行相應的設置，並且允許後台服務啟動視頻通信窗口，然後點擊確定），在打開遠程測試後5分鐘左右，連接建立，可以看到對方的頭像，聽到對方的聲音並還可以發送短消息。

2. 修正了一些錯誤；

1.7.3版

1. 提高了MFP語言的性能。相比1.7.2，MFP總體的速度提高了一倍，運行遊戲速度提高更多。當然，為了確保精度，MFP使用JAVA的BigDecimal類型進行數據計算。而BigDecimal在安卓系統上沒有優化，速度比使用double慢1000倍。這造成了MFP語言的運行速度也比其他語言慢。MFP在未來將引入double數據類型，對於對精度要求不高的計算，使用double類型，將大大提高運行效率；

2. 實現了將MFP示例自動打包為APK的功能；

3. 1.7.3.66增加了一個超級小白兔的游戲；

4. 修正了一些錯誤；

1.7.2版

1. 實現了MFP的二維遊戲引擎。此遊戲引擎還處在初始階段，但已經能夠支持多達70多個函數，能夠靈活使用各種繪圖方法，演奏音樂，以及對滑鼠觸摸屏事件進行偵測。本版本的軟體包括了兩個用MFP語言編寫的小遊戲，一個是削寶石，一個是貪吃蛇。用戶可在探索例子中看到這兩個遊戲。新用戶直接點擊探索示例圖示即可。已經安裝過舊版本的用戶需要先點擊幫助，選擇拷貝示例代碼和圖像，待代碼更新完畢後，點擊探索示例圖示方能看到例子遊戲。用戶可以運行遊戲，也可以閱讀遊戲代碼，還可以把遊戲代碼打包成為apk檔發佈到各大網站供人下載。由於MFP腳本的跨平臺性，MFP遊戲既可以在安卓上運行，也可以在電腦上運行（需要java支持）。在電腦上運行遊戲，一般不會有性能問題。但在手機上運行，如果手機比較舊（超過2年），運行速度會很慢。此外，如果手機螢幕解析度很高，而遊戲沒有注意圖像壓縮，可能會很快消耗掉所有記憶體而崩潰。用戶可以參考削寶石的遊戲代碼看如何壓縮圖像；

2. 引入字典和序列的數據結構。這兩個數據結構都是基於MFP的矩陣。但是和set_array_elem函數不一樣的是，讀寫這兩個數據結構，都是讀寫的引用，而不是原始數值的拷貝。相關的函數是：append_elem_to_ablist，concat_ablists，create_abdict，get_elem_from_ablist，get_value_from_abdict，insert_elem_into_ablist，remove_elem_from_ablist，set_elem_in_ablist以及set_value_in_abdict。用戶可以參考線上幫助獲得詳細資訊；

3. 增加了用於編譯MFP應用的@build_asset注解。該注解可以在打包MFP腳本時自動將圖像聲音檔打入包中。同時修改了@compulsory_link注解的使用方法；

4. 修正了一些錯誤；

1.7.1版

1. 將腳本管理和圖形管理工具合併為單一的檔案管理工具，現在用戶打開檔案管理器，看到的是程式開發科學計算器的根檔案目錄（也就是AnMath檔案目錄）下的所有檔案目錄，但要注意，用戶自定義的腳本檔案必須放在根檔案目錄下的scripts檔案目錄中方能在啟動的時候加載，否則，即便檔案中定義了@execution_entry標註，可以長按運行，運行完之後，也將會被卸載，檔案中的程式將無法在智慧計算器和命令提示符中找到。另外，scripts檔案目錄下的examples子檔案目錄現在被移動到根檔案目錄下的examples下的scripts子檔案目錄了。從1.7.1版開始，scripts檔案目錄下的任何子檔案目錄中的任何mfps腳本檔案都將會被軟體在啟動的時候加載。

3. 增加了gamma程式和beta程式；

4. 修改了derivative和integrate程式的實現方法，現在在計算不定積分和求導數表達式時，輸出的結果僅包括最短的引用空間路徑。在一般情況下，輸出結果只會包含函數名，不會包含引用空間，除非有位於不同引用空間的同名和同定義的函數；

1.6.6版

1. 從1.6.6.50版開始，程式開發科學計算器可以編譯任何檔案名包括漢字的mfps代碼檔案。這意味著，當創建一個APK包的時候，程式開發科學計算器不再拷貝所有用戶自定義的mfps代碼檔案而僅僅抽取相關的代碼。在某些時候，比如調用integrate或者plot_exprs程式時，程式變數是一個字串或者基於字串的變量。這樣一來，程式開發科學計算器在編譯的時候無法判斷哪些函數程式在運行時將會被調用。用戶在這種情況下需要在代碼中，最好在integrate或者plot_exprs程式調用語句的前一行，增加一個註釋指令@complusory_link告知程式開發科學計算器哪些用戶自定義的程式需要鏈接入APK包。比如

...

@compulsory_link function(myf_expr1, myf_expr2(2))

integrated_result = integrate(expression_str, variable_str)

...

在上面例子中，myf_expr1和myf_expr2是用戶自定義的程式。它們在運行時將會被integrate函數程式調用用於計算積分。所有的名字叫做myf_expr1的程式都會被鏈接入APK包。但是對於名字叫myf_expr2的函數程式，當且僅當它正好有兩個變數或者有可選變數時，才會被鏈接入APK包。

如果用戶想把所有自己定義的程式鏈接入APK包，請用如下語句

@compulsory_link all_functions

。這樣一來，程式開發科學計算器將鏈接所有函數程式。但是用戶創建的應用裝載速度會比較慢。並且，用戶還必須保證所有的函數程式都必須已經定義，否則打包時會出現編譯錯誤。

還要註意@compulsory_link指令必須位與一個函數程式的內部，如果它在function語句前面或者endf語句之後，它不會有任何作用。

2. MFP將從1.7.x版開始對set_array_elem函數程式進行一些修改。在當前版本中，set_array_elem將會改變第一個變數的值，哪怕第一個變數只是單個的數而不是陣列。從1.7.x版開始，所有非陣列變數的值將無法在函數程式內部被修改。所以，如果用戶代碼中有如下語句

set_array_elem(x, [1,2], 2+3i)

並且用戶期待x的值在調用該語句後發生改變，請將上述語句修改為

x = set_array_elem(x, [1,2], 2+3i)

在從當前版本開始的1.6.x版中，新的和老的調用方法都可以用，也就是set_array_elem(x, [1,2], 2+3i)和x = set_array_elem(x, [1,2], 2+3i)都能夠正確更新x的值。我們希望這項兼容性能為用戶修改代碼留下充足的時間。

3. 用戶現在可以將任意一個MFP函數程式編譯打包為安卓安裝包（APK檔案）。用戶可以設置APK安裝包的應用名稱，包id，圖標，還可以決定函數程式的哪些變數需要輸入，哪些不需要，以及應用的幫助信息。如果用戶是使用公共簽名文件或者生成公共簽名文件對APK包進行簽名，則可以將APK包在谷歌商店或者其他任何應用發布網站上發布。這項新的激動人心的功能也可以在基於JAVA，運行於個人電腦上的程式開發科學計算器上使用。換句話說，用戶現在可以輕松地在安卓設備或者個人電腦上創建自己的應用。

4. 很多錯誤得到了修正；

1.6.5版

1. 二維圖形表達式繪圖引擎在1.6.5.48中得到顯著改進。由於引入緩存技術以及對拖動事件響應進行了優化，現在計算器對圖形各種操作更加快速和敏感。但要註意，由於對表達式繪圖必須在新的值計算表達式結果，如果表達式過於復雜，用戶仍然會感覺到明顯的延遲。

2. 支持繪制2維或3維隱函數圖形。註意只有當隱函數可解，才能夠繪制圖形。4次或者4次以上的多項式由於根表達式過於復雜，雖然可解，但無法繪圖。。如果繪制包含兩個變數的隱函數，最多4個根表達式將會被繪制出。如果是3個變數的隱函數，根據情況計算器可能會對每個變數求根，然後為每個變數繪制出最多2個解表達式。這樣一來，一個單一的3變數的隱函數，可能意味著繪制出6條圖形，整個求解繪圖過程將會花費比較長的時間。還要註意，隱函數必須是等式，而不能是賦值式，比如x**2+y**2=4-z**2是非法的，而x**2+y**2==4-z**2則是正確的；

3. 函數程式size，zeros和ones的返回值和參數定義有所改變。現在，如果是計算一個數值，而不是陣列的尺寸，size函數程式返回[]而不是[0]。同理，如果參數是[]，zeros返回0而ones返回1；

4. 一定程度的性能提升；

5. 很多錯誤得到了修正；

1.6.0版

1. 更快的計算速度。經過測試，基於JAVA的程式開發科學計算器在電腦上計算速度是以前各版本的10倍，在安卓設備上，由於速度瓶頸未必是在計算上，並且用戶通常一次只運行一個小的命令，對計算速度的提高，用戶感受可能並非非常明顯。但對於大部分計算，速度仍然可以達到以前的1.5到3倍，具體取決於安卓設備的硬體條件；

2. MFP程式開發語言現在可以處理任意大的數，比如，如果計算1000的1000次方，以前的版本會報溢出錯誤，現在用戶可以得到結果為10的3000次方；

3. 計算結果更精確。著名的類Matlab數學軟體Octave采用雙精度數進行內部計算所以其實際精度只能達到約10E-17。但MFP則可以達到5*10E-48。當數值小於這個值時，MFP會視該值為0；

4. 支持無窮大和NaN。由於這項新的特征，本軟體將不會再拋出除0異常。用戶必須自己在代碼中處理。MFP提供了一系列的新程式，比如is_nan_or_inf、is_inf、includes_nan以及includes_inf。使用這些程式用戶可以輕松判斷一個數是否是無窮或者NaN，或者一個復數或者陣列是否包括無窮或者NaN。用戶也可以用系統預定義的變數inf和nan來生成無窮和NaN值並進行計算；

5. 計算特征值和特征向量。註意這是非常消耗內存和CPU資源的計算。強烈建議，在安卓設備上不要計算超過6*6的矩陣，在個人電腦不要計算超過8*8的矩陣。否則可能會內存不足或者用戶陷入長時間等待；

6. 支持自定義智慧計算器和命令提示符的鍵盤。用戶現在不但可以自定義函數程式鍵盤，還可以自定義數字和運算符鍵盤。用戶還可以自由改變按鍵高度，寬度以及按鍵上的字的顏色；

7. 一些其他小的改進和錯誤修正；
